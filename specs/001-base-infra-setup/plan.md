# Implementation Plan: Project Base Infrastructure

**Branch**: `001-base-infra-setup` | **Date**: 2026-06-13 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/001-base-infra-setup/spec.md`

## Summary

Bootstrap the full Next.js 14 App Router project: TypeScript strict, Tailwind CSS, shadcn/ui, brand fonts, Prisma connected to Supabase PostgreSQL, Supabase Storage client, Zod-validated env module, middleware route matcher, and Vercel CI/CD — establishing every foundational layer subsequent features build on.

## Technical Context

**Language/Version**: TypeScript 5.x — strict mode, no `any`, no `as unknown`

**Primary Dependencies**: Next.js 14 (App Router), shadcn/ui, Tailwind CSS, Prisma 5.x, Zod 3.x, next/font (Inter + Playfair Display)

**Storage**: Supabase PostgreSQL (via Prisma ORM) + Supabase Storage (binary uploads)

**Testing**: No test suite in this phase — acceptance validated by running the application and checking acceptance scenarios manually (see [quickstart.md](./quickstart.md))

**Target Platform**: Web — mobile-first responsive, minimum viewport 320px

**Project Type**: SaaS web application (Next.js monorepo, single deployable unit)

**Performance Goals**: First page load on mobile 4G < 3 s (SC-006); startup time with env validation < 1 s overhead

**Constraints**: TypeScript strict with no escape hatches; `process.env` access only through `lib/env.ts`; route protection only in `middleware.ts`; mobile breakpoints as base

**Scale/Scope**: MVP — 1–3 developers, small restaurant customer base; no multi-environment branching required
